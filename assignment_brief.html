Python Intern Assignment

E-Commerce
Data Pipeline

Read two messy datasets, clean them, and produce three output files.

Duration2 days
LevelIntermediate
StackVS Code + pandas

Setup

Create a virtual environment, install pandas, and place both CSV files in your project folder.

python -m venv venv venv\Scripts\activate # Windows source venv/bin/activate # Mac / Linux pip install pandas pip freeze > requirements.txt

Datasets

Both files contain intentional quality issues — part of the assignment is finding them.

orders.csv
200 rows. Sales transactions — product, quantity, price, date, status, country.
customers.csv
50 rows. Customer records — name, email, segment, lifetime value.
Start by loading the data and calling .info() and .isnull().sum() before touching anything.

Tasks

Task 01Load & explore
  • Read both CSVs into DataFrames and inspect them
  • List the data quality issues you find in a comment
Task 02Build a DataCleaner class

The script has a skeleton with docstrings. Implement each method — figure out which pandas tools you need.

  • Standardize text columns (casing, whitespace)
  • Unify country name variants
  • Parse date columns with mixed formats
  • Fill nulls with a strategy that makes sense per column
  • Remove duplicates
  • Add a revenue column (price × quantity × discount)
  • Every method must return self for chaining
Task 03Clean both datasets
  • Run your cleaner on orders, then on customers
  • Print a report before and after each
  • Save to cleaned_orders.csv and cleaned_customers.csv
Task 04Merge
  • Join the two cleaned files on customer_id — keep all orders
  • Save to merged_report.csv
Task 05AnalyseBonus
  • Total revenue per product category
  • Country with the most orders
  • Average order value per customer segment
  • Month with the highest revenue
  • Top 5 customers by total revenue

Deliverables

  • ecommerce_pipeline.py — complete and commented
  • cleaned_orders.csv
  • cleaned_customers.csv
  • merged_report.csv
  • requirements.txt
